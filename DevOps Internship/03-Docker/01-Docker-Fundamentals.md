# Docker Fundamentals

## What is Docker?
Docker is a platform for developing, shipping, and running applications in **containers**.

## Virtual Machines vs Containers
| Feature | Virtual Machine (VM) | Container (Docker) |
| :--- | :--- | :--- |
| **Architecture** | Hypervisor, Guest OS, App | Docker Engine, App |
| **Size** | Heavy (GBs) | Lightweight (MBs) |
| **Boot Time** | Minutes | Seconds |
| **Isolation** | Hardware-level | OS-level |
| **Portability** | Lower | Write Once, Run Anywhere |

## Core Concepts

### 1. Image
A read-only template used to build containers. Contains code, runtime, libraries, etc.
*Analogy: A Class in OOP or a Recipe.*

### 2. Container
A runnable instance of an image. Isolated from other containers and the host.
*Analogy: An Object in OOP or the Meal cooked from the recipe.*

### 3. Dockerfile
A text file containing instructions to build a Docker image.

### 4. Docker Hub / Registry
A cloud repository to store and share Docker images (like GitHub for code).

### 5. Volumes
Mechanism to persist data generated by containers. Data outlives the container.

## Docker Architecture
-   **Client**: Command line tool (`docker`)
-   **Daemon**: Server process (`dockerd`) managing objects
-   **Registry**: Stores images (e.g., Docker Hub)

## Benefits for DevOps
-   **Consistency**: Works on my machine = Works in production
-   **Isolation**: Apps don't conflict
-   **Scalability**: Rapid deployment and scaling
-   **Microservices**: Perfect for distributed architectures
