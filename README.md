# 🚀 30 Days of DevOps Mastery: Zero to Cloud Architect

Welcome to my **DevOps & AWS Learning Journey**. This repository is a dedicated space where I document my transition from an absolute beginner to an aspiring DevOps Engineer. 

Over the next 30 days, I am following a high-intensity, structured roadmap based on the **Code with Mosh DevOps Curriculum**, covering everything from Linux internals to Cloud Architecture.



## 🎯 Project Objective
To master the "Source-to-Server" workflow by building a solid foundation in the five pillars of DevOps:
1. **Infrastructure** (Linux & Networking)
2. **Automation** (Python & Bash)
3. **Cloud** (AWS Core Services)
4. **Containerization** (Docker & Kubernetes)
5. **Observability** (Monitoring & Logging)

---

## 📅 The 30-Day Sprint Roadmap

### 🟦 Week 1: Linux & Networking Core (The Foundation)
*Goal: Move from the GUI to the CLI.*
- **Days 1-3:** Terminal basics (`ls`, `cd`, `mkdir`, `grep`), File System Hierarchy, and VIM/Nano.
- **Days 4-5:** Permissions (`chmod`, `chown`), User Management, and Sudoers.
- **Days 6-7:** Networking: IP Addressing, DNS, Ports (80, 443, 22), and SSH Key Management.

### 🟨 Week 2: Version Control & Programming (The Automation)
*Goal: Stop doing things manually.*
- **Days 8-10:** Git Workflow: Branching, Merging, Pull Requests, and Merge Conflict Resolution.
- **Days 11-14:** Python for Ops: Automation scripts using `os`, `sys`, and `subprocess` modules.

### 🟧 Week 3: Cloud (AWS) & Containers (The Infrastructure)
*Goal: Build scalable, isolated environments.*
- **Days 15-17:** AWS Essentials: IAM (Identity Access), EC2 (Compute), S3 (Storage), and VPC (Networking).
- **Days 18-21:** Docker: Writing `Dockerfiles`, managing volumes, and Docker Compose orchestration.

### 🟪 Week 4: CI/CD & Advanced Ops (The Pipeline)
*Goal: Automate the entire deployment lifecycle.*
- **Days 22-24:** GitHub Actions: Creating CI/CD pipelines (`.github/workflows`).
- **Days 25-27:** Infrastructure as Code (IaC): Introduction to Terraform and State management.
- **Days 28-30:** **Final Capstone:** Auto-deploying a 1-Tier App to AWS using GitHub Actions.

## 🚀 Capstone Project Preview

**Deployment Pipeline Architecture:**
1. Code push to `main` branch.
2. GitHub Actions runs Unit Tests.
3. Docker image is built and pushed to AWS ECR.
4. Terraform provisions an EC2 instance.
5. The application is deployed via a Docker container on AWS.
