# Docker Interview Questions & Answers

## Beginner
1.  **What is a Docker Container?**
    *A lightweight, standalone, executable package that includes everything needed to run a piece of software (code, runtime, system tools, libraries, settings).*

2.  **Difference between Image and Container?**
    *Image is the blueprint (class); Container is the running instance (object).*

3.  **What is a Dockerfile?**
    *A text document that contains all the commands a user could call on the command line to assemble an image.*

4.  **How to check running containers?**
    *`docker ps` lists running containers; `docker ps -a` lists all containers.*

## Intermediate
5.  **Explain the Docker Architecture.**
    *Client-Server architecture. The Docker Client talks to the Docker Daemon, which does the heavy lifting of building, running, and distributing your Docker containers.*

6.  **What is a Docker Volume?**
    *Volumes are the preferred mechanism for persisting data generated by and used by Docker containers. They exist outside the container's lifecycle.*

7.  **What is `ENTRYPOINT` vs `CMD`?**
    *`CMD` sets default command/parameters that can be overridden from the CLI. `ENTRYPOINT` configures a container that will run as an executable (harder to override).*

8.  **How to clean up unused Docker resources?**
    *`docker system prune` removes stopped containers, unused networks, and dangling images.*

## Advanced
9.  **What is Docker Compose?**
    *A tool for defining and running multi-container Docker applications using a YAML file.*

10. **What are Docker Namespaces?**
    *A Linux feature that isolates resources like PID, Network, Mount, User, etc., providing the isolation containers need.*

11. **What is a Multi-stage Build?**
    *A method in Dockerfiles to drastically reduce image size by using multiple `FROM` instructions. You build artifacts in one stage and copy only the necessary binary to the final stage.*

12. **How do containers communicate?**
    *Through Docker Networks. By default, they can talk via the `bridge` network using IP addresses. Creating a custom network allows DNS resolution by container name.*
